# Review Checklist

## Content Completeness

- frontmatter 是否存在且字段齐全。
- 标题是否清晰且与最终结果一致。
- 开头是否说明目标与产出。
- `核心概念` 是否只保留必要术语。
- `你需要准备的内容` 是否包含关键前提与时间。
- 步骤是否从起点推进到最终结果，没有跳步。
- `下一步` 与 `故障排除` 是否存在且有价值。

## Formatting Standards

- frontmatter 是否满足目标项目 schema。
- 步骤标题是否统一为 `## Step N: 动作 + 结果`。
- 步骤内是否用 bullet，而不是继续套编号。
- 代码块、反引号、粗体是否一致。
- `title` 是否过长，`description` 是否过长。

## Executability

- 前提条件是否足够。
- 命令、路径、文件名是否完整。
- 平台、界面操作和顺序是否能照做。
- 是否缺关键判断条件、关键分支或关键产物检查。

## Technical Accuracy

- 命令和配置是否正确。
- 链接和平台入口是否仍有效。
- 版本号、参数名、部署方式是否过时。
- frontmatter 的日期、语言、标签是否与目标项目兼容。

## Fidelity To The Real Process

- 如果存在 `summaryDir/<slug>.md`，教程是否覆盖了摘要中的关键步骤、决策和坑。
- 是否把未发生过的最佳实践写成了实际步骤。
- 是否删掉了会影响复现的真实坑或真实限制。

## Reporting Format

- 先给总体评价，再给分类问题。
- 问题尽量短句，直接指出风险。
- 修复建议要能直接执行。
